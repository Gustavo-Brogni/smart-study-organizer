Modelo de Projetos
	Os Projetos de Sistemas ERP (Enterprise Resource Planning, traduzindo fica Planejamento de recursos da empresa ou Gestão de Administração Empresarial)
	O que os profissionais de desenvolvimento constroem para ser aplicado numa organização, este possui algumas etapas do processo, que podem ter alguns procedimentos. E um destes profissionais são os programadores, outro são os analistas de sistemas. Onde estes procedimentos são divididos em ETAPAS:

Preparação e levantamento dos dados
Definição dos requisitos dos usuários
Criação e/ou desenvolvimento
Uso e (testes de qualidade de softwares)
Implantação (para os clientes) e montagem
Manutenção (Correções, ajustes, configurações)

Preparação e levantamento dos dados
	Na etapa inicial que é a mais importante do projeto, onde os profissionais que geralmente são os Analistas de Sistemas buscam as informações de como funcionam os trabalhos (as tarefas) e o que é usado.

Definição dos Requisitos dos Usuários
	Neste etapa os profissionais então fazem perguntas (quis) ou usam algumas destas informações para determinar como o sistema irá proceder para resolver os problemas (que antes poderiam ser manuais) e acharem as soluções automatizadas para isso, e nesta etapa haverá um contato com os usuários (clientes) para então definir como será esta transição.

Criação e/ou Desenvolvimento
Nesta fase, os desenvolvedores começam a construir o sistema com base nos requisitos definidos anteriormente. Isso envolve a codificação do software, onde as funcionalidades são implementadas de acordo com as especificações. Os desenvolvedores utilizam linguagens de programação e ferramentas de desenvolvimento para criar a aplicação. Além disso, é comum que haja a criação de protótipos ou versões iniciais do software, permitindo que os usuários testem e forneçam feedback antes da versão final. Essa interação é crucial para garantir que o produto atenda às expectativas e necessidades dos usuários.
Uso e Testes de Qualidade de Softwares
Após a criação do software, a próxima etapa é a realização de testes de qualidade. Esses testes são fundamentais para identificar e corrigir erros ou falhas no sistema antes de sua implantação. Existem diferentes tipos de testes, como testes funcionais, de desempenho e de segurança, que garantem que o software funcione corretamente em diversas condições. O feedback dos usuários durante essa fase é valioso, pois ajuda a refinar o sistema e a garantir que ele atenda aos requisitos estabelecidos. A documentação dos testes realizados também é importante para futuras manutenções e atualizações.

Implantação (para os Clientes) e Montagem
A implantação é o processo de disponibilizar o software para os usuários finais. Isso pode envolver a instalação do sistema em servidores, a configuração de ambientes de produção e a migração de dados de sistemas antigos, se necessário. Durante essa fase, é essencial garantir que os usuários recebam treinamento adequado para utilizar o novo sistema. A montagem pode incluir a integração do software com outros sistemas existentes, assegurando que todas as partes funcionem em harmonia. A comunicação clara com os clientes é vital para garantir uma transição suave.

Manutenção (Correções, Ajustes, Configurações)
Após a implantação, a manutenção do software se torna uma prioridade. Isso envolve a correção de bugs que possam surgir, ajustes para melhorar a performance e configurações para atender a novas necessidades dos usuários. A manutenção pode ser dividida em manutenção corretiva, que lida com problemas identificados, e manutenção evolutiva, que busca adicionar novas funcionalidades ou melhorias ao sistema. É importante que haja um suporte contínuo para os usuários, garantindo que eles possam relatar problemas e receber assistência conforme necessário. Essa fase é crucial para a longevidade e eficácia do software.


Classificação de Manutenções
	As manutenções podem ser classificadas em quatro categorias básicas:

Preventivas; 
Corretivas;
Adaptativas;
Perfectivas.

### **1. Modelo de Projetos ERP: Análise Técnica e Profissionais Involvididos**
#### **Conceitos Clássicos e Modernos**
- **ERP** (Enterprise Resource Planning) é um sistema integrado que gerencia recursos como:
  - **Finanças** (contabilidade, orçamento),
  - **Recursos Humanos** (cadastro de funcionários, folha de pagamento),
  - **Produção** (MRP – Material Requirements Planning),
  - **Vendas/Compras** (gestão de pedidos, estoque),
  - **Suporte ao Cliente** (CRM).

*Exemplo prático*: Em uma fábrica de móveis, um ERP pode automatizar desde a fabricação até o atendimento ao cliente, evitando duplicidade de dados entre departamentos.

#### **Etapas com Detalhes Técnicos**
| Etapa               | Descrição Técnica                                                                                     |
|---------------------|-------------------------------------------------------------------------------------------------------|
| **Preparação/Levantamento** | Usa técnicas como:
  - **Entrevistas** (Questionários estruturados com usuários).
  - **Observação direta** de processos (ex: mapear fluxos de vendas em uma loja).
  - **Diagramas** (Fluxograma, ARIS – *Advanced REengineering Information System*). |
| **Definição de Requisitos** | Ferramentas:
  - **Usabilidade**: Testes com protótipos (ex: *User Experience Design* – UXD).
  - **Priorização**: Método MoSCoW (Must-have, Should-have, Could-have). |

#### **Rol dos Profissionais**
- **Analista de Sistemas**: Responsável pela **requisição** e **especificação** das necessidades.
- **Programador**: Implementa a lógica no código (ex: *Java + MySQL*).
  - *Exemplo*: Um analista pode identificar que o sistema precisa registrar notas fiscais, enquanto um programador desenvolve uma API para integração com contadores.

### **2. Desenvolvimento de Projetos Java + MySQL**
#### **Estrutura de Pacotes (Layered Architecture)**
A organização por camadas é fundamental para modularidade e manutenção:

```plaintext
br.com.turma101/
├── DAL        (Data Access Layer)
│   └── Conexao.java          # Gera conexão com MySQL (ex: ConnectionPool)
├── DAO         (Data Access Object)    # Abstração para consultas CRUD
│   ├── UserDAO.java           # Métodos estáticos para operações no banco
│   └── ProdutoDAO.java        # Exemplo de DAO para tabela "produtos"
├── Bean       (Modelo de Dados)
│   ├── UsuarioBean.java
│   │   public int getIdUsuario(); // Getters/Setters
│   └── ProdutoBean.java
├── Telas      (UI)
│   ├── MenuPrincipal.java     # JFrame com layout GridLayout
│   └── CadastroUsuario.java  # Exemplo de tela GUI
└── icones/    # Imagens para botões (ex: "user.png")
```

#### **Explicação Técnica de `Conexao.java`**
```java
public class Conexao {
    private static Connection conn; // Atributo estático para persistência

    public static Connection getConnection() throws SQLException {
        String driver = "com.mysql.cj.jdbc.Driver";  // Versão MySQL Connector
        String url = "jdbc:mysql://localhost:3306/cadastrodb";
        String user = "root";
        String password = ""; // Deve ser configurado conforme ambiente

        conn = DriverManager.getConnection(url, user, password);
        return conn;
    }

    public static void fecharConexao() throws SQLException {
        if (conn != null) conn.close();
    }
}
```
*Detalhes técnicos importantes*:
- **Conector MySQL**: Usar o JAR `mysql-connector-java-8.0.x.jar` para Java.
- **Porta 3306**: Porta padrão do MySQL. Se não funcionar, verifique:
  - *XAMPP/WAMP* está rodando (acesso via `http://localhost/phpmyadmin`).
  - Credenciais do banco estão corretas.

#### **Exemplos Práticos de DAO**
```java
// Exemplo de UserDAO.java
public class UserDAO extends DAO<UsuarioBean> {
    @Override
    public void inserir(UsuarioBean usuario) throws SQLException {
        String sql = "INSERT INTO usuarios (nome, email) VALUES (?, ?)";
        try (Statement stmt = Conexao.getConnection().createStatement()) {
            PreparedStatement pstmt = stmt.prepareStatement(sql);
            pstmt.setString(1, usuario.getNome());
            pstmt.setString(2, usuario.getEmail());
            pstmt.executeUpdate();
        }
    }
}
```
*O que é um DAO?*
- **Data Access Object**: Abstração para operar com banco de dados sem exponuir lógica de conexão.
- *Benefício*: Separa a camada de apresentação (GUI) da camada de dados.

### **3. Testes e Qualidade do Software**
#### **Tipos de Testes e Ferramentas**
| Tipo               | Descrição                                                                                     |
|--------------------|----------------------------------------------------------------------------------------------|
| **Testes Funcionais**   | Verificar se a funcionalidade atende aos requisitos (ex: "O sistema cadastra um usuário"). |
| **Testes de Performance** | Usar *JMeter* ou *Locust*: Simula carga no servidor para evitar falhas.                     |
| **Testes de Segurança**  | Encriptação de dados, autenticação forte (*OAuth*, *LDAP*).                                  |

#### **Exemplo Prático de Testes**
- **Teste Unitário**: Usar *JUnit* para validar métodos DAO.
```java
@Test
public void testInserirUsuario() throws SQLException {
    UsuarioBean usuario = new UsuarioBean("João", "joao@example.com");
    UserDAO dao = new UserDAO();
    dao.inserir(usuario);
    assertTrue(Conexao.getConnection().createStatement()
                 .executeQuery("SELECT COUNT(*) FROM usuarios").getString());
}
```

#### **Ferramentas Recomendadas**
- **MySQL WorkBench**: Para criar tabelas e visualizar dados.
  - *Exemplo*: Criar uma tabela `produtos` com colunas: `id (INT PRIMARY KEY)`, `nome VARCHAR(100)`.
- **NetBeans IDE**: Suporta autocompletar códigos Java e integração com MySQL.

### **4. Implantação e Manutenção**
#### **Implantação**
- **Migração de Dados**: Exportar dados do banco antigo para o novo ERP (ex: via `mysqldump`).
  ```bash
  mysqldump -u root -p antigo_banco > backup.sql
  ```
- **Treino de Usuários**: Utilizar *demonstrações interativas* ou *manuales técnicos*.

#### **Manutenção**
| Tipo          | Descrição                                                                                     |
|---------------|----------------------------------------------------------------------------------------------|
| **Correctiva**| Fixar bugs após implantação (ex: erro no login).                                               |
| **Adaptativa**| Atualizar funcionalidades para novas normas (ex: *LEI DO E-SOCIAL*).                           |
| **Perfectiva**| Melhorias contínuas (ex: novo painel de dashboards).                                           |

#### **Classificação das Manutenções**
- **Preventiva**: Ajustes planejados antes que ocorra um problema (ex: otimizar consultas no banco).
  - *Ferramenta*: *SQL Explanation* para analisar consultas lentas.
- **Perfectiva**:
  - *Exemplo*: Implementar *automação de relatórios* via *JasperReports*.
